# DShop - Часто задаваемые вопросы

## ОБЩИЕ ВОПРОСЫ

### Что такое DShop?
DShop — это модульный плагин для WordPress, который позволяет быстро создавать интернет-магазины. Плагин разработан как универсальный инструмент для студии разработки.

### Зачем нужен DShop?
- Для быстрого создания магазинов для разных заказчиков
- Для унифицированного подхода к e-commerce проектам
- Для снижения времени разработки
- Для обеспечения качества и безопасности

### Какие возможности есть у DShop?
- Каталог товаров (категории, атрибуты, фильтры)
- Корзина и оформление заказа
- Платежные системы (ЮKassa, CloudPayments)
- Доставка (самовывоз, городская, СДЭК, BoxBerry, Почта России)
- Управление запасами
- CRM клиентов
- Скидки и промо
- Email маркетинг
- SEO оптимизация
- Аналитика
- Уведомления (Telegram, VK, SMS)

## ТЕХНИЧЕСКИЕ ВОПРОСЫ

### Какие требования к серверу?
- WordPress 5.9+
- PHP 7.4+ (рекомендуется 8.1+)
- MySQL 5.7+ или MariaDB 10.3+
- Рекомендуется: Redis/Memcached для кэширования

### Как установить DShop?
1. Скачать плагин
2. Закачать в папку `wp-content/plugins/`
3. Активировать плагин в админке
4. Настроить нужные модули

### Как подключить платежные системы?
1. Перейти в Настройки → DShop → Платежи
2. Выбрать нужную платежную систему
3. Ввести API ключи
4. Включить тестовый режим (для проверки)
5. Проверить работу

### Как настроить доставку?
1. Перейти в Настройки → DShop → Доставка
2. Включить нужные методы доставки
3. Настроить параметры каждого метода
4. Установить стоимость и условия

### Как добавить товары?
1. Перейти в DShop → Товары → Добавить новый
2. Заполнить данные товара
3. Загрузить изображения
4. Установить цену и остатки
5. Опубликовать

## АРХИТЕКТУРНЫЕ ВОПРОСЫ

### Почему модульная архитектура?
- Гибкость — включаем только нужные модули
- Производительность — не загружаем лишнее
- Расширяемость — легко добавлять новые модули
- Поддержка — проще поддерживать отдельные модули

### Как добавить новый модуль?
1. Создать папку в `modules/`
2. Создать главный класс модуля
3. Реализовать интерфейс `ModuleInterface`
4. Зарегистрировать модуль в `Loader.php`
5. Добавить настройки модуля

### Как переопределить шаблон?
1. Создать папку `dshop/` в теме WordPress
2. Скопировать шаблон из `templates/default/`
3. Изменить шаблон по необходимости
4. DShop автоматически использует ваш шаблон

### Как добавить хук?
```php
// Действие
do_action('dshop/my_action', $data);

// Фильтр
$value = apply_filters('dshop/my_filter', $value);
```

## ПРОИЗВОДИТЕЛЬНОСТЬ

### Как ускорить магазин?
1. Включить кэширование (Redis/Memcached)
2. Оптимизировать изображения
3. Использовать CDN
4. Минифицировать CSS/JS
5. Включить gzip сжатие

### Сколько товаров выдержит DShop?
- До 10000 товаров — отлично
- До 50000 товаров — хорошо (с оптимизацией)
- Более 50000 товаров — нужна дополнительная оптимизация

### Как проверить производительность?
1. Использовать Query Monitor
2. Проверять время загрузки страниц
3. Мониторить запросы к БД
4. Использовать New Relic или аналоги

## БЕЗОПАСНОСТЬ

### Какие меры безопасности применены?
- Nonce verification для всех форм
- Capability checks для админских действий
- Sanitization и validation входных данных
- SQL injection protection через prepared statements
- XSS protection через escaping
- CSRF protection

### Как обновлять плагин?
1. Скачать новую версию
2. Заменить файлы плагина
3. Проверить работу
4. При необходимости запустить миграции БД

### Как делать бэкапы?
1. Бэкап файлов плагина
2. Бэкап базы данных
3. Бэкап загрузок
4. Регулярно проверять бэкапы

## ПРОБЛЕМЫ

### Плагин не активируется
1. Проверить версию PHP (7.4+)
2. Проверить версию WordPress (5.9+)
3. Проверить наличие зависимостей
4. Посмотреть логи ошибок

### Товары не отображаются
1. Проверить статус товара (опубликован)
2. Проверить настройки видимости
3. Проверить шаблоны
4. Очистить кэш

### Платежи не работают
1. Проверить API ключи
2. Проверить тестовый режим
3. Проверить логи платежей
4. Связаться с поддержкой платежной системы

### Доставка не рассчитывается
1. Проверить настройки доставки
2. Проверить адрес доставки
3. Проверить вес и размеры товара
4. Проверить зоны доставки
