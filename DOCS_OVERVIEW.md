## ПОРЯДОК ИСПОЛЬЗОВАНИЯ ДОКУМЕНТОВ

### 1. При начале проекта:
1. Прочитать **README.md** для общего понимания
2. Изучить **ARCHITECTURE.md** для понимания структуры
3. Ознакомиться с **REQUEST.md** для понимания требований
4. Следовать **ACTION_PLAN.md** для начала работы

### 2. При разработке:
1. Использовать **FILE_TREE.md** для навигации
2. Следовать **TECH_REQUIREMENTS.md** для написания кода
3. Консультироваться с **FAQ.md** при возникновении вопросов
4. Контролировать прогресс по **PLAN.md**

### 3. При тестировании:
1. Использовать **LAUNCH_CHECKLIST.md** для проверки
2. Сверяться с **ARCHITECTURE.md** для проверки соответствия

### 4. При запуске:
1. Пройти **LAUNCH_CHECKLIST.md**
2. Проверить все пункты из **PLAN.md**
